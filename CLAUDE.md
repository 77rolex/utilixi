## О проекте

**Название:** utilixi.com
**Тип:** Multi-tool utility сайт (сборник бесплатных онлайн-инструментов)
**Цель:** Пассивный доход через Google AdSense после набора трафика
**Домен:** utilixi.com (куплен на Hostinger)
**Хостинг:** Vercel (бесплатный Hobby план)

---

## Стек

- **Framework:** Next.js 16.2.6 (App Router) / React 19.2.4
- **Стили:** SCSS модули (без Tailwind)
- **Мультиязычность:** next-intl ^4.12.0 — 5 локалей: en, ru, uk, fr, lt
- **Email:** nodemailer (Gmail SMTP)
- **Аналитика:** Vercel Analytics + Speed Insights + Google Analytics
- **Хостинг:** Vercel (Hobby план, автодеплой из GitHub)
- **Домен:** Hostinger → DNS указывает на Vercel

## Архитектурные решения

### Навигация
- Каждая страница — отдельный инструмент, минимум внешних ссылок
- Хедер (десктоп): логотип + кнопка «Инструменты» → выпадающее меню с горизонтальной строкой категорий, при наведении на категорию — список инструментов в столбик по 5
- Хедер (мобайл): логотип + бургер → сайдбар с аккордеоном по категориям
- Футер: инструменты по категориям через `<details>/<summary>` (без JS) + служебные страницы

### Полный список страниц

Служебные (только в футере, не в хедере):
```
/[locale]/about          — о проекте (обязательно для AdSense)
/[locale]/contact        — контакт (обязательно для AdSense)
/[locale]/privacy-policy — политика конфиденциальности (обязательно для AdSense)
```

Технические (без locale):
```
/404          — страница не найдена (кастомная)
/500          — серверная ошибка (кастомная)
/sitemap.xml  — генерируется автоматически Next.js
/robots.txt   — генерируется автоматически Next.js
/             — редирект на /en/ (или по языку браузера)
```

### База данных
- Не нужна для большинства инструментов
- Калькуляторы — чистый JS
- Погода/валюты — внешние API
- Гороскоп — Claude API + серверный кэш на день

### API принципы
- Использовать только бесплатные API
- Кэшировать ответы где возможно (Next.js revalidate)
- Никогда не хранить API ключи в коде — только .env.local

## SEO требования (важно!)

- Каждая страница: уникальный title, meta description, h1
- hreflang теги для всех языков на каждой странице
- sitemap.xml — генерируется автоматически Next.js
- robots.txt — разрешить всё
- Schema markup (JSON-LD) для каждого инструмента
- Под каждым инструментом — текстовое описание + FAQ (для AdSense одобрения)
- Mobile-first дизайн (60-65% трафика с мобильных)

## Важные договорённости

1. Делаем по одному инструменту за раз
2. Сначала делаем — деплоим — следующий инструмент
3. SEO разметка делается сразу, не потом
4. Каждый инструмент: текстовое описание + FAQ под виджетом
5. Литовские переводы генерирует Claude
6. Не использовать платные API
7. Не использовать Tailwind
8. БЭМ именование классов — строго везде
9. Все цвета, отступы, шрифты — только через CSS переменные
10. Тёмная тема — заготовка уже в переменных, реализация позже
11. Советы и предложения по улучшению — приветствуются
12. Связанные инструменты — добавлять перекрёстные ссылки через компонент `RelatedTools` в конце страницы.
13. **Git — только вручную пользователем.** Claude никогда не выполняет `git add`, `git commit`, `git push` самостоятельно. Все коммиты делает только разработчик.
    Группы: Финансы (ипотека ↔ кредит), Здоровье (BMI ↔ калории), Крипта (курсы ↔ конвертер).

14. **Sitemap — обновлять после каждой сессии.** После добавления нового инструмента — обязательно добавить его URL в `src/app/sitemap.ts` (PAGES массив, с нужным priority и changeFrequency). Без этого новая страница не попадёт в индекс Google.

## Правила разработки (обязательно соблюдать)

### 1. Мультиязычность — всегда и везде
- Сайт на **5 языках: en, ru, uk, fr, lt** — любое изменение затрагивает все 5
- Переводить нужно **всё**: кнопки, лейблы, placeholder, сообщения об ошибках, подсказки, пустые состояния, aria-label
- Литовский (lt) — нет носителя, генерирует Claude; проверять на логичность
- Никогда не оставлять хардкод строк на одном языке в JSX

### 2. Чистота и производительность кода
- При изменении — **удалять старый ненужный код** (мёртвые переменные, неиспользуемые импорты, устаревшие стили)
- Не дублировать логику — выносить в shared-модули если используется в 2+ местах
- Минимум зависимостей — не тянуть библиотеку ради одной функции
- Кэшировать API-ответы через `next: { revalidate: N }`, не делать лишних запросов
- Изображения — только через `next/image` (оптимизация WebP, lazy load автоматически)

### 3. SEO — на всех языках, для каждого инструмента
- Уникальные `title`, `meta description`, `h1` для каждой страницы × каждый язык
- `hreflang` альтернативы на всех страницах (уже реализовано через `routing.locales`)
- JSON-LD разметка (Schema.org `WebApplication`) на каждом инструменте
- Описательный текст + FAQ под каждым виджетом (требование AdSense + SEO)
- Canonical URL прописывать явно где нужно
- Внутренняя перелинковка между связанными инструментами через компонент `RelatedTools` — обязательна для SEO

### 4. Кроссбраузерность и кроссплатформенность
- Поддержка: Chrome, Firefox, Safari, Edge (последние 2 версии)
- Платформы: Android, iOS, macOS, Windows, Linux
- iOS Safari: избегать `position: fixed` без хака, проверять `-webkit-` префиксы где нужно
- Не использовать экспериментальные CSS-свойства без fallback
- Touch-события: минимальный tap target — **44×44px** (рекомендация Apple/Google)
- Проверять что hover-эффекты не ломают UX на touch-устройствах (`:hover` vs `:focus`)

### 5. Mobile-first — в первую очередь
- Все стили пишутся сначала для мобильных, десктоп — через `@media (min-width: ...)`
- Точки перелома: `640px` (sm), `768px` (md), `1024px` (lg)
- На мобильных скрывать второстепенные колонки таблиц (класс `--hide-mobile`)
- Шрифты, отступы, кнопки — проверять на экране 375px (iPhone SE) как минимум
