# Utilixi

Free online utility tools — calculators, converters, and more.
Live site: [utilixi.com](https://utilixi.com)

---

## Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 16.2.6 (App Router, Turbopack) |
| Language | TypeScript |
| Styles | SCSS Modules (no Tailwind) |
| i18n | next-intl ^4.12.0 |
| Email | Nodemailer + Gmail SMTP |
| Analytics | Vercel Analytics + Speed Insights + Google Analytics |
| Hosting | Vercel (Hobby) |
| Domain | Hostinger → DNS → Vercel |

---

## Languages

5 languages via the `[locale]` route segment:

| Code | Language |
|---|---|
| `en` | English (default) |
| `ru` | Russian |
| `uk` | Ukrainian |
| `fr` | French |
| `lt` | Lithuanian (generated by AI, no native speaker) |

URLs follow the pattern `/en/calculator/mortgage`, `/ru/calculator/mortgage`, etc.

```tsx
  <PageLayout sidebar={<AdSidebar />}>
    <h1>{meta.h1}</h1>
    <ToolWidget locale={locale} />
    <AdInline locale={locale} />                   {/* Mobile only */}
    <div className={styles.page__content}>
      <p>{content.description}</p>
      <RelatedTools locale={locale} tools={related} />  {/* + desktop ad banner inside */}
      <section className={styles.faq}>...</section>
    </div>
  </PageLayout>
</>
```

## Translations

All translations are **inline** in each `page.tsx` — there is no `messages/` directory.
Every page defines `META`, `CONTENT`, and `RELATED` objects with keys `en | ru | uk | fr | lt`.
Widget components receive `locale` as a prop and use their own internal translation objects.

---

## SEO

Every tool page includes:
- Unique `<title>`, `<meta description>`, `<h1>` per locale
- `hreflang` alternates for all 5 languages (via `buildAlternates()` in `src/lib/seo.ts`)
- `x-default` pointing to `/en/`
- JSON-LD `WebApplication` schema markup
- `<h1>` + descriptive text + FAQ section (required for AdSense approval)
- `sitemap.xml` with hreflang `alternates.languages` for every locale entry
- Mobile-first layout

## Ad slots

| Component | Location | Visible on |
|---|---|---|

| `AdInline` | After widget, before content | Mobile + tablet <1024px |
| `AdPlaceholder` (banner) | Inside `RelatedTools` + home page | Desktop only (banner) |

## Contact form

- Cloudflare Turnstile CAPTCHA (server-side verification)
- Honeypot field against basic bots
- In-memory rate limiting: 3 requests / 5 min / IP
- Sends via Gmail SMTP (nodemailer)

## Design conventions

- **SCSS Modules** — one `.module.scss` per component/page
- **BEM naming** — strictly `block__element--modifier`
- **CSS variables only** — no hardcoded colors, spacing, or font sizes (see `globals.scss`)
- **Dark theme** — variables declared under `[data-theme='dark']`, not yet activated
- **Mobile-first** — breakpoints: 640px / 768px / 1024px
- **No UI libraries** — no Tailwind, no MUI, no shadcn
